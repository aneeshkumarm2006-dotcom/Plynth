## 1. Telemetry + error pipeline (foundation)
**Migration:** `0009_telemetry_tables.sql`

What changed:
- Two append-only tables: `telemetry_events` (page views, key actions, timings)
  and `error_events` (exceptions / failed RPC/Supabase calls, with `severity` +
  `fingerprint` grouping). Enums `telemetry_app`, `event_severity`.
- **Asymmetric RLS:** any authenticated user INSERTs only their own rows; only
  admins SELECT. No UPDATE/DELETE policy → rows immutable except the retention
  purge (runs as table owner).
- Write choke point `ingest_telemetry(p_events, p_errors)` (SECURITY DEFINER,
  granted to `authenticated`): forces `user_id := auth.uid()` (no spoofing),
  caps `message`/`stack`, strips route query strings, computes the fingerprint,
  bulk-inserts a whole batch in one round trip.

Decisions: telemetry is **product-analytics grade, not legal** — `audit_log`
stays the 7-yr PIPEDA record. Retention is 90d events / 180d errors (`0012`).

Test points:
- ☐ `[sec]` RLS insert-own / admin-read; client-sent `user_id` ignored by the RPC.
- ☐ `[sec]` PII guardrail: no query strings / borrower PII land in stored rows.

## 2. Client telemetry SDK
**New:** `packages/supabase/src/telemetry/{index,core,session}.ts` +
`@plynth/supabase/telemetry` export; wired in all three apps' `main.tsx`.

What changed:
- Public API: `initTelemetry`, `track`, `trackTiming`, `trackPageView`,
  `captureError`, `flush`. Batched (flush at 20 items / 5s / tab-hide /
  pagehide), **fails silently** (never throws into app code), auto-captures
  `window error` + `unhandledrejection`.
- **Strict PII allowlist** in `session.ts`: props reduced to scalar values, PII-
  named keys dropped, strings truncated, ≤12 keys.
- **Mock mode = pure no-op** — no queue, no network — so the 308-test suite and
  demo mode are unaffected.
- Admin app emits `page_view` via a `RouteTracker` on every route change.

Test points:
- ☐ `[int]` Signed-in navigation → `page_view` rows; thrown error → `error_events`.
- ⚠️ Mock mode must stay no-op or the unit suite breaks (every SDK fn guards on `live()`).

## 7. Automated evaluation + retention
**Migration:** `0012_alert_eval_cron.sql`

- `evaluate_alert_rules()` checks every enabled rule (5 kinds) respecting
  `cooldown_min`, opens `alert_events` on breach; `admin_run_alert_eval()` is the
  admin-gated "Run now" wrapper. Seeds 5 default rules if none exist.
- **pg_cron:** evaluation every 5 min + nightly purges (telemetry 90d / errors
  180d / alert_events 1yr). Defensive — if pg_cron is unavailable the functions
  still install (NOTICE) and "Run now" is the manual path.

Test points:

- ☐ Retention purges run; `audit_log` never purged.
